## 規格書

為使專案「有露點的肉番福利動漫列表」未來能順利進行，特訂定此規格書 (Specification) 。

內容規範了製作表格的流程，動漫的收錄標準，還有遵循事項。

「有露點的肉番福利動漫列表」約是從2021年11月開始製作，架構經過多次變更。現參照世界趨勢所訂定規格，可能會不定時修正。

**這個規格書為理想情況，是作為未來執行的參考。實務上現存的資料仍有多處需要修正的地方。**


### 動漫媒介限制

收錄自1917年以來播映的動漫，但是我最早只能找到1970年代的，還有一些1960年代的手塚治虫動畫。

此列表收錄的項目：
- 電視動畫（表番）
- OVA/OAD/特典
- 遊戲或漫畫改編的動畫
- 泡麵番（例如 彗星社 製作的動畫）
- 動畫電影

此列表不收的項目：
- 成人動畫（裏番）
- 遊戲內的動畫
- 同人動畫（例如《東方Project》）
- 非日本製作的動畫
- 藝術祭的實驗作品
- 真人版戲劇


### 關於露點與肉番的認定標準

雖然這個列表叫做露點動漫，並多次提及肉番，但是偏向的是真正有露點的動畫，並不是肉番就一定會露點。

且因為賣肉鏡頭的動畫太多，如果把露內褲和泳衣都算進去的話根本數不完，故以露點為第一要件。

僅有一集或一小段露點的也會收錄在此列表。

簡言之，動畫必須有真的露點才可以收進列表，肉番要素僅是加分條件。

這些算露點：
- 角色必須是女性，MTF也可以，不限物種，但起碼得是人形
- 露點指的是明確的描繪出胸部乳頭的畫面
- 光線很暗下的露點（例如《機巧少女不會受傷》）
- 激突，衣服明確露出乳頭輪廓
- 露出乳暈（《下流梗不存在的灰暗世界》S01E04 安娜·錦之宮）
- 濕掉衣服所透露出的乳頭（《嗜血狂襲》S01E02 姫柊雪菜）
- 裸露且刻畫出乳頭輪廓（《刀劍神域 序列爭戰》亞絲娜）
- 畫得很隨便的乳頭（《我的女友是個過度認真的處女bitch》OVA01 香坂秋穂）

這些不算露點：
- 露內褲
- 露出內衣
- 露出屁股
- 露出腿部
- 露出腋下...總而言之乳頭以外的個人性癖部位都不算露點
- 入浴或溫泉畫面
- 開黃腔
- 性暗示手勢
- 真實存在的藝術圖畫而非角色本身露點
- 有聖光或異物遮掩的乳房畫面
- 性行為姿勢但只有描繪大略的身體畫面
- 大面積裸露但沒有描繪乳頭（《女神咖啡廳》S01E01 小野白菊）
- 描繪全身裸體但沒有描繪乳頭（《遊戲人生》的超健全空間、《美少女戰士》的聖光變身畫面）
- 獵奇
- BL場景
- 男性裸體

如果覺得有很棒的肉番，但是沒露點無法收錄進此列表，那麼可以放到「沒露點的實用肉番動漫列表」作回收。


### 特殊判例

有些收錄於列表的動畫或OVA，其實算裏番，例如泡麵番，還有許多西元2000年以前的動畫。

### 動漫片源

如果是當季新番，請至少等所有集數播放完再拍攝。

片源以BD為佳，通常要在動畫播映幾個月後才會發售。

第一優先使用P2P軟體到Nyaa找10-bit 1080p BDRip (Blu-ray) 片源下載，通常BD就是無修正 (uncensored) 版本。

如果可以的話自購正版BD轉檔。

次之為串流 (WEB-DL) 片源，例如巴哈姆特動畫瘋、Bilibili，或是其他串流網站的片源。如果因為趕新番而使用了串流片源，建議日後要換回BD片源。

不要使用TV版 (HDTV) 片源，多半會遮擋得很兇。

### 動漫表格欄位資料

動漫資料以JSON記載（下述）

所有的動漫資料最終會渲染成一個HTML表格。

表格中，水平的格子稱作列，一列即為一部動漫。

表格中，垂直的格子稱作欄，有這四個欄位：

1. 動漫名稱，需要包含三種語言的翻譯
2. 播映年份，需要具體到年月日
3. 露點集數與預覽截圖，截圖需包含集數與角色資訊，需要包含三種語言的翻譯
4. 解說與吐槽，需要包含三種語言的翻譯。可不填。

如果動漫有很多季，將每季動畫和OVA都獨立成一列。若有多部OVA也建議分開表示。

這不是動漫百科，請只列出有露點的季數。因為就算是肉番動漫也不是每季都會露點。

例如：
```
- IS〈Infinite Stratos〉
- IS〈Infinite Stratos〉第二季
- IS〈Infinite Stratos〉
- IS〈Infinite Stratos〉 OVA Encore 愛情戰火六重奏
- IS〈Infinite Stratos〉 OVA 分離解體的各個世界
```

動漫第一季不需要特別寫「第一季」。動漫第二季若有另外取名，優先採用，不稱作第二季。

例如：
```
- 彼得·格里爾的賢者時間
- 彼得·格里爾的賢者時間Super Extra
```

動漫的中文翻譯使用台灣譯名，如果台灣沒出版再使用中國的，或是網路上的慣用譯名。

動漫的英文翻譯以日文羅馬字為主（不含長短音符號），看情況使用意譯譯名。

動漫播映日期要具體到月和日（並非BD發售時間），以日本時間為主。


### 拍攝露點集數的預覽截圖

由於版面關係，每部動畫的預覽圖以三張為限，截圖不可以包含字幕。

2000年以後的動畫盡量使用10-bit 1080p BDRip片源。

截圖原則上不拍攝OP與ED的畫面。三張截圖最好來自不同集數且不同角色。

儘管很多時候其他畫面更養眼，請以真的有露點和角色臉蛋的畫面優先，才符合本列表的標準。

角色優先鎖定主角。優先拍攝露點部份，並有露出角色臉蛋的為佳。次之為拍攝露點但只有乳房的視角。可視該作品強調的性癖部份，斟酌拍攝屁股、陰部截圖。

如果真的只有一幕露點，剩餘兩張用很肉但沒露點的場景截圖來補足。如果真的沒有其他很肉的場面，那就一張就好。

截圖使用WEBP儲存。原始檔為PNG的話，轉檔成WEBP的時候使用90%品質。

檔名格式：
```
動漫名稱_季數_集數_角色名稱.副檔名
```

動漫、角色名稱之間的空格使用橫線填滿。檔案字元只可以包含英文數字，不可以包含Linux和Windows檔案系統不允許的特殊符號。

「季」使用S表示，若數字小於10需要在前面補0。

「集」使用E表示，若數字小於10需要在前面補0。

「特別篇Special」使用SP表示，若數字小於10需要在前面補0。

沒有特別名字的「劇場版」使用Movie表示。

「OVA」或「OAD」和其他名稱使用原名表示，若數字小於10需要在前面補0。

例如：
```
IS-Infinite-Stratos_S01E06_Charlotte-Dunois.webp
```

如果截圖有多個角
參見：[Specification.md](Specification.md)
色，使用底線分隔。只需要標註女性角色名稱。

例如
```
IS-Infinite-Stratos_S01E08_Charlotte-Dunois_Laura-Bodewig.webp
```

檔案取名使用日文羅馬字拼音（不含長短音符號），順著寫，不使用英文格式的倒轉姓氏。

如果三張圖片都是同一個角色且同一集數：圖片最後面加上_1、_2、_3。

圖片註解字串需要包含三國語言，並以HTML的`<title>`和`<alt>`屬性渲染在圖片上。

參見：[Specification.md](Specification.md)

為方便手機版用戶閱讀，另外渲染成p屬性在圖片下方。


### JSON檔架構

通常JSON是不允許`//`註解的，雖然有JSONC標準存在，但Hugo編譯不會通過。因此註解的形式改用JSON的一個數值代替（不會渲染到實際網頁）。下面的註解`//`只是為了方便說明。

以10年為一個單位進行分割（1970~1979、1980~1989，剩餘的以此類推）。這樣做的理由是同一個網頁塞全部年份內容太多，若按照每一年下去切割內容又太少，故以十年為一個單位。此外這樣也方便觀察不同年代的畫風變化。

拆開成8個json檔案和網頁。
```bash
├── fanserviceanimelist-1960-1969.json
├── fanserviceanimelist-1970-1979.json
├── fanserviceanimelist-1980-1989.json
├── fanserviceanimelist-1990-1999.json
├── fanserviceanimelist-2000-2009.json
├── fanserviceanimelist-2010-2019.json
├── fanserviceanimelist-2020-2029.json
```

每個json檔案的最外層為一個陣列
```json
[

]
```

陣列內含多個物件，每個物件表示一部動漫。
```json
[
	{
	//動漫1
	},
	{
	//動漫2
	}
]
```

## 專案進度

- 將所有列表的預覽圖補完。
- 逐步替換掉avif格式的舊圖片。
- 每部動畫都寫個解說，吐槽劇情，講解看點。
